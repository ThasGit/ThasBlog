Java基础知识

Posted on 2019-10-28 | In 基础知识
  • 面向对象的特性
    面向对象有三大特性：封装、继承、多态。
  1. 封装是指将特定的实现细节通过访问权限控制的形式“隐藏”起来，调用者只需关注怎么用即可。
  2. 继承是指某个类可以通过extends关键字继承其父类的全部成员，以实现代码复用和功能扩展的目的。继承是多态的前提。
  3. 多态就是指多种形态，同一个方法可以根据不同的子类实现呈现出不同的功能，即重写；同一个对象可以在其继承链上进行类型转换，以适应目标场景的需求或减少对自身信息的暴露。一般情况下也会将方法重载视为重载的一部分。
  4. 抽象是指将某一类事物的共同特征提取出来，而无需实现，这些抽象特征将具有更宽泛的使用场景，而具体实现交给具体的实现类去实现。
  • 访问权限修饰符
    共有public、protected、default、private四个访问权限。
  1. public是完全的公开权限，任何地方都可以访问。
  2. protected是受保护的访问权限，同包全部类以及不同包下的子类可以访问。
  3. private是私有权限，仅本类或本类的内部类可以访问到。
  4. 不写修饰符，非接口代表仅同包下的类可以访问，接口内的成员不管写不写修饰符都是public访问权限

  • 为什么有clone
    clone是对于引用类型而言的，一个引用对象不管经过怎样的参数传递，其代表的都是堆上的同一个实例，如果想要复制出一个额外的实例，则必须新创建一个实例，并赋予与原对象相等的成员值。clone方法可以实现这种效果。

  • new和clone的区别

  1. new会先在内存上开辟空间，然后执行构造方法，通过构造方法对类中的数据域填充新值，最后将该对象的引用地址返回给外部。
  2. clone方法同样也是开辟一段新的内存空间，然后复制原对象的数据域的内容到该数据域，最后将该对象的引用地址返回给外部。该复制操作是JVM自动进行的。

  • 深拷贝和浅拷贝的区别
    两者区别是相对于拷贝类中的引用类型而言的，JVM自动实现的clone方法，在复制引用类型的成员变量的时候，复制的只是引用地址，两个实例的相同引用类型成员变量指向同一份成员实例，称之为浅拷贝。深度拷贝需要手动实现，针对类中的引用成员变量一一操作，手动clone出新的引用成员实例，然后绑定给成员变量。

  • java中的goto关键字
    java保留了goto关键字，但是并没有实现。java中的break可以实现跳转语句的效果，但仅限于循环语句。

  • hashcode和equals
    hashcode可以看做equals的低配版本，hashcode相同，equals不一定true；hashcode不同，equals一定为false。hashcode必须保证自反性、对称性、传递性和一致性。

  • String是否可以继承
    String是final类型，不可以被继承。

  • Java中的引用类型传参
    两种观点：

  1. 引用类型传参属于引用传递。传递的参数可以对原实例做修改。
  2. 不属于引用传递。java中传递引用类型时实际上是实例引用的值传递。
  • 方法重写和重载的区别
  1. 方法重写是指子类对父类已有的方法进行新的实现，从而替换掉原有的方法，对父类该方法的调用将会链接到子类重新实现的方法上。重写的必要前提是继承。
  2. 方法重载是编译时多态，指可以定义多个方法名相同，但参数列表不相同的方法。

  • 方法重写的规则
    方法名相同，参数列表完全相同，返回值相同或更加精确，异常列表相同、更少或更加精确，访问范围相同或更大。

  • 为什么返回值类型不能用来区分重载方法
    调用方法时是不能指定返回类型的，因此调用时不能区分返回值不同的方法。比如一个带返回值的方法，是可以不接收其返回值直接调用的。

  • char类型能够存储一个汉字
    java中char统一使用Unicode编码保存字符，所以可以保存汉字。

  • 抽象类和接口区别
    抽象类和类的区别仅仅在于，抽象类可以包含抽象方法，不可以实例化。所以抽象类和接口的区别完全可以看做类和接口的关系，它们没有半毛钱的关系。都是用来表示抽象的概念，抽象类是对象本质的抽象，通常用于存放公共属性和方法；接口用来表示对象所具备的功能。

抽象类可以包含属性、实现或未实现的方法、构造方法，可以实现接口，继承类，只能单继承。

接口只能包含抽象方法、常量、静态方法。

  • 抽象方法是否可以是static、native、synchronized
  1. 抽象方法不可以是static的，static与实例无关，不支持运行时多态，不能通过子类来实现或者重写。
  2. 抽象方法不能是native的，native方法是JVM自动实现的，不是由Java子类实现的。
  3. 可以synchronized，synchronized修饰静态方法，表示锁定该类的Class对象。

  • 静态变量和实例变量的区别
    静态变量与实例无关，当类加载到内存，并完成初始化之后，静态变量就能直接被使用，可以理解为类的所有实例的共享变量。
    实例变量在类实例化时创建，只能通过创建它的实例访问到。

  • ==和equals区别

  1. ==可以比较值类型或引用类型，判断值类型是否相等或引用类型是否指向同一实例。
  2. equals判断两个引用类型是否是“相等”，但默认情况，如果不对equals实现，其功能同==。
  • break和continue区别
  1. break退出当前层的循环，加标签表示退出标签所表示层的循环。
  2. continue退出当前层的当次循环，加标签表示退出标签所示层的当次循环。

  • Java实现多态的机制
    通过使用父类型的变量指向子类的实例实现，父类的方法的实现取决于实例化的子类。

  • Java异常分类
    Java异常大体分为受检查异常和运行时异常。

  1. 受检查异常：异常必须要显式处理，或抛出交给调用者处理。该种异常通常无法通过优化编码解决，但可以忽略或恢复，对后续运行不产生影响。
  2. 运行时异常：异常可以不进行显式处理，该种异常一般可以通过优化编码的方式避免发生，比如空指针异常和索引超界异常。
  3. Error通常不归类到异常，Error是严重的错误，无法通过程序解决。
